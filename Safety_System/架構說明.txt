一、 系統架構總結
本系統採用層次分明的模組化設計，主要分為四大層級：
1.	引導與環境層 (Bootstrapping)：處理程式啟動、單一執行體檢查及環境適應（High-DPI）。
2.	核心資料層 (Data Access Layer)：負責所有數據的持久化儲存（純文字檔讀寫）與執行緒安全控制。
3.	主導覽外殼 (Main Shell)：系統的主視窗，負責菜單管理、快捷鍵攔截及動態載入功能模組。
4.	功能模組層 (Business Modules)：獨立的業務邏輯頁面，如巡檢、報表、設定等。
二、 程式碼檔案對應說明
1. 專案配置與環境
• Safety_System.csproj
• 說明：專案的「身分證」。定義了編譯目標（.NET 4.7.2）、輸出格式（WinExe）以及專案名稱。它決定了程式如何被打包與編譯。
• app.config
• 說明：環境設定檔。最關鍵的功能是啟動 PerMonitorV2 的 High-DPI 支援，修正了 WinForms 在高解析度螢幕下字體模糊的問題。
2. 核心邏輯
• Program.cs
• 說明：程式的進入點。
• 關鍵功能：
• 使用 Mutex 實作 防止程式重複啟動。
• 啟動時呼叫 DataManager.LoadConfig() 預載資料路徑。
• DataManager.cs
• 說明：系統的「資料庫管理員」。
• 關鍵功能：
• 路徑動態化：管理 BasePath，支援使用者自定義資料存放資料夾。
• 執行緒安全 (Thread-Safety)：使用 lock 關鍵字確保在多工作業下寫入文字檔不會產生衝突。
• 資料格式化：以 | 作為欄位分隔符號進行讀寫。
3. 主介面控制
• MainForm.cs
• 說明：系統的「主戰情室」。
• 關鍵功能：
• 1440 x 810 佈局：定義主視窗大小與 UI 字體清晰度。
• 動態模組切換 (LoadModule)：點擊選單時，動態替換中間 Panel 的內容，無需切換多個視窗。
• 快捷鍵攔截：透過 ProcessCmdKey 實作 Ctrl + 3 快速跳轉功能。
• 選單架構：整合「首頁」、「報表」、「工安」等導覽。
4. 業務功能模組 (獨立 .cs 檔案)
• App_SafetyInspection.cs
• 說明：工安巡檢功能模組。實作了巡檢地點、人員、狀態的輸入介面與資料儲存邏輯。
• App_DbConfig.cs
• 說明：資料庫路徑設定視窗。
• 關鍵功能：實作為一個 Modeless (非互動式) 視窗，允許使用者在不關閉主視窗的情況下同時進行設定，支援資料夾選取與路徑持久化。
• App_Report.cs
• 說明：報表中心。目前預留了「月報表」與「年報表」的容器空間，用於未來的數據分析與圖表展示。
